## 实验原理

### 1. 质点-弹簧模型（Mass-Spring Model）

质点-弹簧系统是计算机图形学中最经典的变形体模拟方法之一。我们将布料离散化为网格状的质点集合，质点之间通过弹簧相连。

根据胡克定律（Hooke's Law），两个质点 a 和 b 之间的弹力公式为：

```
f_a = -ks * (|x_a - x_b| - l) * (x_a - x_b) / |x_a - x_b|
```

其中 `ks` 为弹簧的劲度系数，`l` 为弹簧的原长，`x` 为质点位置。

同时，为了防止系统能量无限增加导致发散，需要引入阻尼力（Damping force）：

```
f_d = -kd * v_a
```

### 2. 弹簧类型

| 类型 | 连接方式 | 示意图 | 作用 |
|------|---------|--------|------|
| 结构弹簧 | (i,j)-(i+1,j), (i,j)-(i,j+1) | 水平和垂直边 | 抵抗拉伸/压缩 |
| 剪切弹簧 | (i,j)-(i+1,j+1), (i+1,j)-(i,j+1) | 对角线 | 抵抗剪切形变 |
| 弯曲弹簧 | (i,j)-(i+2,j), (i,j)-(i,j+2) | 隔一个质点 | 抵抗弯曲形变 |

### 3. 数值积分方法

根据牛顿第二定律，质点的加速度 `a = F/m`。在离散的时间步 `dt` 内，通过数值积分更新质点的速度 `v` 和位置 `x`：

- **显式欧拉（Explicit Euler）**：完全使用当前时刻的状态来预测下一时刻。
  ```
  x_{t+1} = x_t + v_t * dt
  v_{t+1} = v_t + a_t * dt
  ```
  优点：计算简单。缺点：能量不守恒，容易数值爆炸。

- **半隐式欧拉（Semi-Implicit / Symplectic Euler）**：先更新速度，再使用更新后的速度更新位置。
  ```
  v_{t+1} = v_t + a_t * dt
  x_{t+1} = x_t + v_{t+1} * dt
  ```
  优点：比显式欧拉更稳定。缺点：仍然是显式方法，大步长时不稳定。

- **隐式欧拉（Implicit / Backward Euler）**：使用未来时刻的状态计算受力，使用定点迭代法近似求解。
  ```
  v_{t+1} = v_t + a_{t+1} * dt
  x_{t+1} = x_t + v_{t+1} * dt
  ```
  优点：无条件稳定。缺点：需要迭代求解，计算开销大。

### 4. 防爆处理

- **速度钳制**：限制质点的最大速度 `max_velocity`，防止在显式欧拉等不稳定方法中出现严重的数值爆炸。
- **边界约束**：限制布料最低高度，防止布料无限下坠。

### 5. 球体碰撞（选做）

在物理更新时判断布料质点与球体的距离。如果质点在球体内部，则：
1. 将质点推出球体表面
2. 反转速度的法向分量（带弹性系数）

## 实现内容

### 任务 1：场景初始化（GPU 同步）

按照实验要求，将初始化操作拆分为多个 `@ti.kernel`，在 Python 侧按顺序调用：

- `init_positions()`：初始化 20x20 质点的位置、速度和受力，标记顶部两行为固定点
- `init_springs()`：初始化结构弹簧、剪切弹簧和弯曲弹簧的拓扑结构
- `init_render_indices()`：初始化三角形渲染索引（每个正方形格拆分为两个三角形）

### 任务 2：力学计算与防爆处理

- `compute_spring_forces()`（`@ti.func`）：计算重力、弹簧力，使用 `ti.atomic_add` 避免多线程写入冲突
- `clamp_velocity()`（`@ti.func`）：限制质点最大速度，防止数值爆炸

### 任务 3：积分求解器实现（核心）

三个 `@ti.kernel` 分别实现三种积分方法：

- `step_explicit()`：显式欧拉，`x_new = x + v*dt`，`v_new = v + a*dt`
- `step_semi_implicit()`：半隐式欧拉，`v_new = v + a*dt`，`x_new = x + v_new*dt`
- `step_implicit()`：隐式欧拉，固定点迭代 5 次逼近

受力计算和位置/速度更新合并在同一个 Kernel 中完成，最小化每帧循环内的 Kernel 启动次数。

### 任务 4：渲染与 GGUI 交互

使用 Taichi 的 `ti.ui.Window` 构建 3D 场景，包含：

- **场景渲染**：质点（蓝色小球）+ 弹簧（灰色线段）+ 碰撞球体（棕色粒子群）
- **GUI 控制面板**：
  - 积分器切换：显式欧拉 / 半隐式欧拉 / 隐式欧拉
  - 暂停/继续按钮
  - 重置布料按钮
  - 阻尼系数调整：1.0 / 5.0 / 10.0
  - 选做功能开关：剪切弹簧 / 弯曲弹簧 / 球体碰撞

### 选做内容 1：完善弹簧模型

在结构弹簧的基础上，补充了剪切弹簧和弯曲弹簧：

- **剪切弹簧**：连接对角质点，抵抗剪切形变，使布料更"硬"
- **弯曲弹簧**：连接隔一个的质点，抵抗弯曲形变，防止布料过度折叠

可在 GUI 中独立开关剪切弹簧和弯曲弹簧，观察不同弹簧组合对布料形态的影响。

### 选做内容 2：球体碰撞

在场景中放置一个球体（棕色粒子群），在物理更新时判断布料质点与球体的距离：

- 如果质点在球体内，将其推出球体表面
- 反转速度法向分量（带弹性系数 1.2，产生微小反弹效果）
- 碰撞检测与响应集成在积分 Kernel 中，无需额外 Kernel 启动

## 注意事项

- **Taichi 版本**：建议使用 Taichi 1.6+ 版本，确保 GGUI 功能完整。
- **GPU 推荐**：本实验依赖 CUDA 加速，无 GPU 时自动回退到 CPU，但运行速度较慢。
- **固定点**：布料顶部两行质点被固定，模拟布料悬挂效果。可修改 `pinned` 逻辑改变固定方式。
- **隐式欧拉的迭代次数**：增加迭代次数可提高精度，但会降低帧率。当前设置为 5 次，在精度和性能间取得平衡。
- **子步循环**：每帧执行 5 次物理更新（子步），确保在低帧率下模拟依然稳定。

- **GUI 文字**：Taichi GGUI 不支持中文字体，GUI 标签均使用英文。
- **碰撞球体默认关闭**：点击 GUI 中的 "Collision: OFF" 按钮可开启球体碰撞。
